## üìã Project Description

**Riggioni's Gallery** is a modern web application designed to showcase and manage a visual artist's digital portfolio. The platform allows visitors to explore artworks, get detailed information about each project, add projects to favorites, and contact the artist directly through WhatsApp.

This project was developed as part of an **innovative educational program** that aims to bring students closer to a real work environment through a **Code Review model** led by a developer acting as a "client". This pedagogical approach has the following objectives:

- ‚úÖ **Improve code quality** generated by students
- ‚úÖ **Simulate professional environments** in software development
- ‚úÖ **Foster good practices** in programming from early stages
- ‚úÖ **Develop communication skills** in technical contexts
- ‚úÖ **Learn to work with feedback** constructive and continuous

const toggleFavorite = async (projectId: string, projectName: string) => {

## üîß Installation and Setup

### Prerequisites
- Node.js >= 18.x
- npm or yarn
- Firebase account (Firestore + Remote Config)
- Vercel account (for deployment)

### Local Installation

1. **Clone the repository**
```bash
git clone https://github.com/rooseveltalej/RiggionisGallery.git
cd RiggionisGallery
```

2. **Install dependencies**
```bash
npm install
```

3. **Configure environment variables**

Create `.env.local` file:
```env
VITE_FIREBASE_API_KEY=your_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_auth_domain
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_storage_bucket
VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
VITE_FIREBASE_APP_ID=your_app_id
```

4. **Configure Firebase Remote Config**

In Firebase console, create `languages` parameter with structure:
```json
{
  "languages": {
    "spanish": {
      "navbar": { "routes": { /* ... */ } },
      "gallery_page": { "projects": [ /* ... */ ] },
      "general_titles": { /* ... */ },
      "artist_content": { /* ... */ }
    },
    "english": { /* ... */ }
  }
}
```

5. **Run in development**
```bash
npm run dev
```

6. **Build for production**
```bash
npm run build
```

## üéì Educational Approach: Code Review Model

This project was developed under an **innovative pedagogical approach** where a developer acts as a "client" and performs constant code reviews to:

### Benefits for Students

1. **Real-Time Feedback**
   - PR reviews with constructive comments
   - Architectural improvement suggestions
   - Early detection of anti-patterns

2. **Best Practices from the Start**
   - Consistent naming conventions
   - Separation of concerns
   - Strict typing with TypeScript
   - Effective componentization

3. **Professional Environment Simulation**
   - Git Flow usage
   - Pull requests with detailed descriptions
   - Manual testing and feature validation
   - Code documentation

4. **Soft Skills Development**
   - Effective technical communication
   - Handling constructive criticism
   - Remote collaborative work
   - Merge conflict resolution

### Applied Methodology

```
Development Cycle:
1. Student implements feature
2. Opens Pull Request with description
3. Code Reviewer (client) reviews and comments
4. Student refactors based on feedback
5. Approval and merge
6. Continuous iteration
```

### Quality Metrics Evaluated

- ‚úÖ Code readability
- ‚úÖ Maintainability and scalability
- ‚úÖ Performance and optimizations
- ‚úÖ Accessibility (a11y)
- ‚úÖ Responsiveness
- ‚úÖ Error handling
- ‚úÖ Security (env vars, sanitization)
